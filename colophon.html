How I Built This

Architecture decisions, tradeoffs, and why I chose this stack.

What This Site Does

A professional portfolio with a working contact form. No server, no database, no backend maintenance.

When someone submits the contact form, it verifies they're human (Cloudflare Turnstile), sends me an email (Resend API), and responds instantly. All serverless.

Tech Stack & Why

Frontend

Pure HTML, CSS, vanilla JavaScript. No framework, no build step. Faster load times, zero dependencies, works everywhere. For a portfolio, React would be overkill.

Hosting: Cloudflare Pages

Free, global CDN, auto-SSL, GitHub integration. Push to main ‚Üí live in 30 seconds. 300+ edge locations mean <50ms response times worldwide. Alternative was Vercel or Netlify, but Cloudflare's integration with Workers sealed it.

Backend: Cloudflare Workers

Serverless functions at the edge. No servers to maintain, scales automatically, runs in <10ms. Handles form submissions, validates CAPTCHA, triggers emails. Cost: $0 (100k requests/day free tier).

Email: Resend API

Reliable, developer-friendly, 100 emails/day free. Tried MailChannels first (free but unreliable). Resend has better deliverability, proper SPF/DKIM setup, and clean API. Emails arrive in seconds, not spam folders.

Spam Protection: Cloudflare Turnstile

Privacy-friendly CAPTCHA. No Google reCAPTCHA tracking. Smart challenges only when needed. Free, integrates natively with Workers.

Architecture Flow

1. User submits form
2. Turnstile validates (not a bot)
3. Worker receives POST request
4. Worker calls Resend API
5. Email lands in my inbox
6. User sees success message

Total time: ~500ms

Why document this? Senior architects make infrastructure decisions daily. This shows how I think: evaluate options, understand tradeoffs, optimize for the use case. A portfolio doesn't need Kubernetes.
