How I Built This

Technical decisions and why I chose this stack.

What This Site Does

This is a static portfolio with a serverless contact form. No servers to manage, no database to maintain.

When someone fills out the contact form, it verifies they're human using Cloudflare Turnstile, then sends me an email through Resend's API. The whole flow happens instantly at the edge.

Tech Stack

Frontend

Pure HTML, CSS, and vanilla JavaScript. No framework, no build step. For a portfolio site, React felt like overkill. The entire JS bundle is under 20KB, and pages load in under a second even on slower connections.

Hosting: Cloudflare Pages

Connected to GitHub for automatic deployments. Push to main and the site goes live in about 30 seconds. Cloudflare's global CDN means fast response times from anywhere, and the integration with Workers made it the obvious choice over Vercel or Netlify.

Backend: Cloudflare Workers

Serverless functions that run at the edge. Handles form submissions, validates the CAPTCHA token, and triggers the email. No servers to maintain, scales automatically, and executes in milliseconds.

Email: Resend API

Initially tried MailChannels but the deliverability was inconsistent. Switched to Resend for better reliability and proper SPF/DKIM configuration. Emails arrive in seconds and don't end up in spam folders.

Spam Protection: Cloudflare Turnstile

Chose this over Google reCAPTCHA to avoid tracking users. Turnstile is privacy-friendly and only shows challenges when necessary. Integrates cleanly with Workers.

Architecture Flow

1. User submits form
2. Turnstile validates (not a bot)
3. Worker receives POST request
4. Worker calls Resend API
5. Email lands in my inbox
6. User sees success message

Total time: ~500ms

I documented this because infrastructure decisions are a core part of system architecture work. Choosing the right tools for the job matters, and a portfolio site doesn't need Kubernetes.
